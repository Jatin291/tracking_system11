Introduction

Project Overview

The Employee Management System is a comprehensive solution designed to streamline HR processes, automate attendance tracking, and manage employee leave requests efficiently.

Project Goals:

  • Simplify employee attendance tracking
  • Automate leave request management
  • Provide real-time analytics and reporting
  • Enhance communication between employees and HR
  • Ensure data security and privacy

Technical Highlights:

Node.js Express MongoDB JWT Auth REST API

API Documentation

Authentication Endpoints

🔑 User Registration

POST /api/register

Request Body:

{ "username": "string", "password": "string" }

Response (201):

{ "message": "User registered successfully" }

🔐 User Login

POST /api/login

Request Body:

{ "username": "string", "password": "string" }

Response (200):

{ "message": "Login successful", "role": "user|admin", "username": "string", "email": "string", "token": "jwt.token.here" }
4/8

API Documentation

Leave Management Endpoints

📝 Submit Leave Request

POST /api/leave/request

Headers: Authorization: Bearer [token]

Request Body:

{ "leaveType": "casual|sick|vacation", "startDate": "YYYY-MM-DD", "endDate": "YYYY-MM-DD", "reason": "string" }

📋 Get Leave History

GET /api/leave/history

Headers: Authorization: Bearer [token]

Response (200):

[ { "_id": "string", "leaveType": "string", "startDate": "date", "endDate": "date", "status": "pending|approved|rejected", "reason": "string" } ]

API Documentation

Attendance Endpoints

⏱️ Clock In

POST /api/attendance/clock-in

Headers: Authorization: Bearer [token]

Response (200):

{ "message": "Clocked in successfully", "clockInTime": "timestamp" }

⏰ Clock Out

POST /api/attendance/clock-out

Headers: Authorization: Bearer [token]

Response (200):

{ "message": "Clocked out successfully", "workingHours": { "hours": 8, "minutes": 30, "seconds": 0 } }

API Documentation

Admin Endpoints

👥 Get All Users

GET /api/admin/users

Headers: Authorization: Bearer [admin_token]

Response (200):

{ "users": [ { "_id": "string", "username": "string", "email": "string", "role": "user|admin" } ] }

✅ Update Leave Status

PUT /api/leave/status/:id

Headers: Authorization: Bearer [admin_token]

Request Body:

{ "status": "approved|rejected" }

API Usage Examples

Example: User Login

fetch('http://localhost:3001/api/login', { method: 'POST', headers: { 'Content-Type': 'application/json', }, body: JSON.stringify({ username: 'user123', password: 'password123' }) }) .then(response => response.json()) .then(data => console.log(data));

Example: Submit Leave Request

fetch('http://localhost:3001/api/leave/request', { method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': 'Bearer your.jwt.token.here' }, body: JSON.stringify({ leaveType: 'casual', startDate: '2025-08-01', endDate: '2025-08-03', reason: 'Family event' }) });
